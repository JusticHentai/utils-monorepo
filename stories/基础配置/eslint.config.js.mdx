## 配置详解

### `import eslintConfig from './packages/config-utils/eslint.config.js'`

- **作用**：从配置工具包中导入 ESLint 配置
- **说明**：
  - 复用 `packages/config-utils` 中的统一 ESLint 配置
  - 避免在根目录重复配置相同的规则
  - 便于统一管理整个 monorepo 的代码规范

### `export default eslintConfig`

- **作用**：导出 ESLint 配置
- **说明**：
  - 直接使用配置工具包中的配置
  - 根目录的 ESLint 配置与所有子包保持一致
  - 确保整个项目的代码风格统一

### 配置来源

根目录的 ESLint 配置继承自 `packages/config-utils/eslint.config.js`，该配置包含：

- **JavaScript 核心规则**：`@eslint/js` 推荐规则
- **TypeScript 规则**：`typescript-eslint` 推荐规则
- **Prettier 集成**：`eslint-plugin-prettier` 集成配置
- **React Hooks 规则**：`eslint-plugin-react-hooks` 规则
- **React Fast Refresh**：`eslint-plugin-react-refresh` 规则
- **浏览器环境**：`globals.browser` 全局变量定义

### 优势

- **统一配置**：整个 monorepo 使用相同的代码规范
- **易于维护**：修改配置工具包即可影响所有项目
- **减少重复**：避免在每个包中重复配置
- **一致性**：确保所有代码遵循相同的规范
