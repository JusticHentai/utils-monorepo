# onLeaveViewport

监听元素离开视口（一次性）。当目标元素首次离开视口时触发回调，触发后自动停止观察。

## 详细介绍

`onLeaveViewport` 是基于 `IntersectionObserver` 封装的一次性视口离开监听工具。元素必须先进入视口后再离开才会触发回调。

**涉及的知识点**：
- IntersectionObserver API：浏览器原生的元素可见性检测 API
- 状态追踪：记录元素是否曾经可见
- 一次性观察：触发后自动停止

**常用场景**：
- 用户离开监测：检测用户是否已滚动过某个重要区域
- 播放器控制：视频元素离开视口时暂停播放
- 广告曝光结束：统计广告从可见到不可见的时间

**优势**：
- 一次性触发，无需手动清理
- 确保元素曾经可见后才触发离开事件
- 比 scroll 事件监听更高效

## 具体实现原理

### 文件结构

- `index.ts` - 主函数实现
- `interface.ts` - 类型定义

### 核心流程

1. 使用 `hasBeenVisible` 变量追踪元素是否曾经可见
2. 当 `isIntersecting` 为 `true` 时，标记 `hasBeenVisible = true`
3. 当 `isIntersecting` 为 `false` 且 `hasBeenVisible` 为 `true` 时，触发回调
4. 触发后调用 `stop()` 停止观察

### 关键技术点

- 使用闭包保存 `hasBeenVisible` 状态
- 确保元素必须先可见后才能触发离开事件
- 一次性触发后自动清理
