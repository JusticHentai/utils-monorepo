# onFCP

监听 FCP (First Contentful Paint)。FCP 是首次内容绘制时间，衡量页面首次渲染内容的速度。

## 详细介绍

`onFCP` 是基于 `PerformanceObserver` 封装的 FCP 性能指标监听工具。

**涉及的知识点**：
- FCP 指标：页面首次绘制文本、图片、非空白 canvas 或 SVG 的时间
- PerformanceObserver API：监听 paint 类型
- 与 LCP 区别：FCP 是首次内容绘制，LCP 是最大内容绘制

**FCP 评分标准**：
- 好：≤ 1.8 秒
- 需要改进：1.8 - 3 秒
- 差：> 3 秒

**常用场景**：
- 性能监控：评估页面首次渲染速度
- 白屏优化：监控白屏时间
- 用户体验：了解用户首次看到内容的时间

**优势**：
- 一次性触发，精确捕获首次内容绘制
- 自动过滤 first-paint 只获取 first-contentful-paint
- 触发后自动停止监听

## 具体实现原理

### 文件结构

- `index.ts` - 主函数实现
- `interface.ts` - 类型定义

### 核心流程

1. 使用 `createPerformanceObserver` 监听 `paint` 类型
2. 从 entries 中查找 `name === 'first-contentful-paint'` 的条目
3. 找到后触发回调并停止监听

### 关键技术点

- paint 类型会返回 first-paint 和 first-contentful-paint 两种
- 需要过滤只获取 first-contentful-paint
- 触发后自动调用 `stop()` 停止监听
