# onEnterViewport

监听元素进入视口（一次性）。当目标元素首次进入视口时触发回调，触发后自动停止观察。

## 详细介绍

`onEnterViewport` 是基于 `IntersectionObserver` 封装的一次性视口进入监听工具。

**涉及的知识点**：
- IntersectionObserver API：浏览器原生的元素可见性检测 API
- 一次性观察：触发后自动停止，避免重复回调
- 阈值配置：支持设置元素可见比例触发条件

**常用场景**：
- 图片懒加载：元素进入视口时加载真实图片
- 曝光统计：统计元素是否被用户看到
- 动画触发：元素进入视口时播放入场动画
- 无限滚动：检测底部元素进入视口时加载更多

**优势**：
- 一次性触发，无需手动清理
- 比 scroll 事件监听更高效
- 支持配置可见比例阈值

## 具体实现原理

### 文件结构

- `index.ts` - 主函数实现
- `interface.ts` - 类型定义

### 核心流程

1. 调用 `createIntersectionObserver` 创建底层观察器
2. 在回调中检测 `isIntersecting` 是否为 `true`
3. 首次进入视口时执行用户回调并调用 `stop()` 停止观察
4. 使用 `immediate: true` 确保立即检测当前状态

### 关键技术点

- 复用 `createIntersectionObserver` 底层实现
- 一次性触发后自动清理，避免内存泄漏
- 空元素安全处理，返回空函数
