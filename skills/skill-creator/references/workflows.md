# 工作流程模式

## 顺序工作流程

对于复杂任务，将操作分解为清晰的顺序步骤。在 SKILL.md 开头给 Claude 一个流程概述通常很有帮助：

```markdown
填写 PDF 表单包括以下步骤：

1. 分析表单（运行 analyze_form.py）
2. 创建字段映射（编辑 fields.json）
3. 验证映射（运行 validate_fields.py）
4. 填写表单（运行 fill_form.py）
5. 验证输出（运行 verify_output.py）
```

## 条件工作流程

对于具有分支逻辑的任务，引导 Claude 通过决策点：

```markdown
1. 确定修改类型：
   **创建新内容？** → 遵循下方"创建工作流程"
   **编辑现有内容？** → 遵循下方"编辑工作流程"

2. 创建工作流程：[步骤]
3. 编辑工作流程：[步骤]
```
